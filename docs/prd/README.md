# Product Requirements Documents (PRDs)

Product requirements for the open-core control plane + safety layer. Each PRD
states the problem and solution from the user's perspective, the implementation
and testing decisions, scope boundaries, and acceptance criteria — then gets
decomposed into one-branch/one-PR issue packets.

| PRD | Title | Status |
|---|---|---|
| [0001](0001-control-plane-and-safety-layer.md) | Control plane + safety layer for Robinhood Agentic Trading (M1 deep; M2–M4 epics) | Draft v2 |

PRDs are numbered sequentially. M2–M4 are captured as epics in PRD 0001 and
should each get their own PRD before build. See `docs/decisions/` for the ADRs a
PRD honors and `docs/spikes/` for open de-risking spikes a PRD depends on.
